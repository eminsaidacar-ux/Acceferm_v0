## Guidance globale pour la post-prod

- Couper tout overlay texte inattendu du modèle en post (OpenAI v2 ajoute parfois du texte parasite).
- Si un visage apparaît malgré le prompt, régénérer — aucun compromis sur la contrainte CLAUDE.md.
- Export AVIF + WebP optimisés < 200 KB par image pour rester sous budget LCP.
- Hotlinker depuis un CDN (Cloudflare Images / Vercel Image Optimization) plutôt que depuis `/public` en prod.
- Ajouter ces images dans `storefront/lib/images.ts` comme nouvelles clés `imagery` lorsque disponibles.

## Roadmap de déploiement

1. Générer les 10 images chez OpenAI v2 avec les prompts ci-dessus.
2. Les placer dans `storefront/public/hero/` (nomenclature : `01-hero.avif`, `02-expertise.avif`, etc.).
3. Étendre `lib/images.ts` pour référencer ces chemins locaux en priorité, fallback Unsplash.
4. Tester Core Web Vitals avec les vrais visuels avant mise en prod.
